# RSSI Localization

This repository contains RSSI-based path localization baselines and a ROS2 package for replay and online inference.

## Repository Structure

```text
code/
+-- robot_s_baseline.py
+-- rssi_fingerprint_baseline.py

    \-- src/

            |   \-- rssi_localizer_node.py

            +-- setup.cfg
            \-- setup.py
```

## Main Components

- `robot_s_baseline.py`
  - Robot-project baseline for predicting global path progress `s` from aggregated RSSI windows.
- `rssi_fingerprint_baseline.py`
  - Earlier fingerprint baseline for path tracking experiments.
- `ros2_ws/src/rssi_localization/`
  - ROS2 package for replaying RSSI CSV data and publishing online `s` estimates.

## Baseline Usage

### 1. Robot `s` baseline

Expected CSV format:

```csv
s,timestamp,beacon_1_mean,beacon_1_std,beacon_2_mean,beacon_2_std,...
```

Run Scheme A:

```powershell
.\.venv\Scripts\python.exe .\robot_s_baseline.py `
  --train .\examples\robot_s_demo\train.csv `
  --test .\examples\robot_s_demo\test.csv `
  --k 3 `
  --max-jump-s 0 `
  --smooth-alpha 0 `
  --out baseline_outputs\robot_predictions_A.csv `
  --map-out baseline_outputs\robot_map_A.csv
```

Run Scheme B:

```powershell
.\.venv\Scripts\python.exe .\robot_s_baseline.py `
  --train .\examples\robot_s_demo\train.csv `
  --test .\examples\robot_s_demo\test.csv `
  --k 3 `
  --max-jump-s 5 `
  --smooth-alpha 0.4 `
  --out baseline_outputs\robot_predictions_B.csv `
  --map-out baseline_outputs\robot_map_B.csv
```

Outputs:

- prediction CSV
- fingerprint map CSV
- optional per-file summary CSV

### 2. RSSI fingerprint baseline

Example:

```powershell
.\.venv\Scripts\python.exe .\rssi_fingerprint_baseline.py `
  --target-mode s `
  --train .\examples\robot_s_demo\train.csv `
  --test .\examples\robot_s_demo\test.csv `
  --out baseline_outputs\smoke_predictions.csv `
  --map-out baseline_outputs\smoke_fingerprint_map.csv
```

See [REAL_S_BASELINE.md](REAL_S_BASELINE.md) and [RSSI_BASELINE.md](RSSI_BASELINE.md) for more details.

## ROS2 Usage

The ROS2 package name is `rssi_localization`.

### 1. Build

From the workspace root:

```bash
cd ros2_ws
colcon build
```

Then source:

```bash
source install/setup.bash
```

On Windows PowerShell, use the corresponding local setup script generated by your ROS2 installation.

### 2. Start the localizer node

The localizer node subscribes to RSSI scans and publishes predicted path progress `s`.

Launch with:

```bash
ros2 launch rssi_localization localizer.launch.py
```

Important parameters in `localizer.launch.py`:

- `radio_map_path`
  - Path to a prebuilt fingerprint map CSV.
- `train_csv_glob`
  - Alternative to `radio_map_path`; build the localizer from training CSV files directly.
- `beacon_order`
  - Beacon order expected in incoming RSSI vectors.
- `input_topic`
  - Default: `/rssi_scan`
- `output_topic`
  - Default: `/path_s_estimate`
- `debug_topic`
  - Default: `/path_localization_debug`
- `start_s`
  - Initial path progress estimate.
- `monotonic_increase`
  - If `True`, the predicted `s` will not decrease.

The node expects:

- input message type: `std_msgs/Float32MultiArray`
- output message type: `std_msgs/Float32`
- debug message type: `std_msgs/String`

### 3. Replay a CSV as RSSI topic input

The package also provides `csv_replay_node`, which reads a CSV file and publishes one RSSI vector per row.

Example:

```bash
ros2 run rssi_localization csv_replay_node --ros-args \
  -p csv_path:=/absolute/path/to/test.csv \
  -p publish_hz:=2.0 \
  -p output_topic:=/rssi_scan
```

Important replay parameters:

- `csv_path`
  - Required path to the replay CSV.
- `publish_hz`
  - Publishing rate.
- `output_topic`
  - Default: `/rssi_scan`
- `beacon_order`
  - Column order used to extract `beacon_*_mean` values.
- `loop_playback`
  - Whether to replay the file repeatedly.

### 4. Typical ROS2 workflow

Terminal 1:

```bash
cd ros2_ws
source install/setup.bash
ros2 launch rssi_localization localizer.launch.py
```

Terminal 2:

```bash
cd ros2_ws
source install/setup.bash
ros2 run rssi_localization csv_replay_node --ros-args \
  -p csv_path:=/absolute/path/to/test.csv
```

Terminal 3:

```bash
ros2 topic echo /path_s_estimate
```

Optional debug stream:

```bash
ros2 topic echo /path_localization_debug
```

## Notes

- The robot `s` baseline uses path progress `s` as the main target.
- Missing RSSI markers such as `-999` are converted internally to a weak RSSI value.
- For ROS2 online use, the beacon order in the incoming topic should match the localizer model order.
- IMU and odom streams can be used as motion priors or temporal constraints around the RSSI fingerprint estimate.
